# Fitness Tracker Skill

一个让 AI 帮你管理身体长期进化的健身 Skill。

大多数健身工具关注：

> 今天练了什么？

而 Fitness Tracker Skill 关注：

> 我的身体正在发生什么变化？下一步应该怎么调整？

它通过记录：

- 训练数据
- 体重变化
- 饮食情况
- 恢复状态
- 主观感受

帮助 AI 理解你的身体状态，并给出长期优化建议。

**核心理念：**

> 身体不是一个需要偶尔修理的机器，而是一套需要持续运营的系统。

## 安装

```bash
git clone https://github.com/PandaDalmore/fitness-tracker-Skill.git
```

克隆后按你的 Agent 的约定放入对应 skills 目录即可。

适用所有支持 Skill 的 AI Agent——Obsidian vault、Cursor 工程、VS Code 项目、独立终端均可。

> 可选：配置 Gemini API Key（照片分析用）。免费获取 https://aistudio.google.com/apikey → `export GEMINI_API_KEY="你的Key"`

## 首次使用

对 AI 说以下任意一句即可触发初始化引导：

- "初始化健身信息"
- "帮我设置健身追踪"
- "算热量"

AI 会引导你填写：性别、年龄、身高、体重、体脂率（可选）、BMR（可选，有实测值最佳）、训练频率、主要目标。可以跳过任何项，系统会用默认估算值并标注提示后续补充。

## 使用流程

### 方式一：对话直接算（快速）

适合碎片化查询，无需打开任何文件：

> 👤 早上两个鸡蛋一杯牛奶，中午鸡腿饭，晚上牛肉面，步数 8000，中午练了 40 分钟力量
>
> 🤖 收到。三层拆解：BMR×1.1=1,766 + NEAT(8000步)=360 + EAT(力量40min)=136，TDEE=2,262。
> 全天摄入估算 ~1,700 kcal，赤字 -562。蛋白 ~95g/109g 🟡，距目标差 14g。

支持的说法：*"算热量""昨天吃了什么帮我看看""分析这张照片""本周复盘"*。

### 方式二：HTML 表单 → AI 分析（完整周追踪）

适合系统化管理一整周的数据，输出可视化报告。

**1. 初始化（首次）**

对 AI 说「初始化健身信息」，按引导填写性别、年龄、身高、体重、目标等。可跳过任意项，系统用默认值标注"估算"。

**2. 打开表单**

对 AI 说「打开周数据追踪表」，AI 会在浏览器中打开 `weekly-tracker.html`。

**3. 设置保存目录**

点击表单顶部「📁 选择目录」→ 选择本地文件夹（如 Obsidian 仓库的健身目录）→ **手动补全完整路径**（如 `E:\笔记\健身`）。目录权限持久化，下次自动恢复。

**4. 每天填数据**

- 7 个 Tab 切换 → 填完即自动保存到浏览器
- **运动**：下拉选类型 + 填时长 → 自动按 `MET × 体重 × 小时` 算消耗
- **步数**：填数字 → 自动算 NEAT（每千步 45 kcal）
- **饮食**：描述食物 + 可选热量（留空让 AI 估算）+ 📷 上传食物照片
- **身体数据**：体重 / 体脂率 / 腰围 / 睡眠
- 底部实时显示当天 摄入 / TDEE / 缺口

**5. 送出分析**

点「📊 日报」或「📈 周报」：

1. 自动保存 MD 文件 + 食物照片到磁盘
2. 结构化数据（`<tracker-data>` JSON）+ 触发词 复制到剪贴板
3. 切回 AI 对话 → **Ctrl+V 粘贴**

**6. AI 处理 & 回填**

AI 收到剪贴板内容后自动：
1. 解析 JSON → 计算 TDEE / 缺口 / 蛋白质
2. 读取食物照片 → 视觉估算热量
3. 注入 HTML 模板 → 生成可视化仪表盘
4. 把估算热量回写到 MD 文件

回到表单 → 点「📥 更新数据」→ 自动解析最新 MD → 热量回填到对应餐次。

```
表单录入 ──→ MD+照片 ──→ 剪贴板JSON ──→ AI分析 ──→ HTML仪表盘
    ↑                                                   │
    └──── 「更新数据」回填 ←── JSON回写 ←─────────────────┘
```

### 食物照片分析

- **Claude 原生视觉**：当前模型支持图片输入时自动使用，零配置
- **Gemini 免费 API**（备选）：日 1500 次免费，[获取 Key](https://aistudio.google.com/apikey) → `export GEMINI_API_KEY="你的Key"`
- ⚠️ 用户文字描述优先于 AI 视觉估算（AI 看图精度 ±30%）
- ⚠️ 中文路径可能损坏，Gemini 模式下自动复制到纯英文路径

## 核心公式 — 三层防御账本

TDEE 分三层独立计算，避免活动重叠：

```
TDEE = BMR×1.1 + 步数×45/1000 + Σ(MET×体重×小时)
```

- **静态保本项（BMR×1.1）**：基础代谢 + 睡眠消化产热。BMR 有实测值用实测值（InBody/DEXA），无实测值用 Mifflin-St Jeor 估算
- **日常非运动消耗（NEAT）**：步数兜底，每千步 45 kcal
- **运动性消耗（EAT）**：MET 国际通用运动强度标准，覆盖力量训练、跑步、骑行、游泳、HIIT 等

热量缺口 = 摄入 - TDEE。公式唯一不分目标，正负号自然区分盈余/赤字。

## HTML 可视化

将每日/每周的健身数据渲染为交互式 HTML 图表页面：

- **每日仪表盘**：TDEE 三层堆叠柱状图、摄入 vs 消耗对比、餐次环形图、蛋白质进度环、运动明细表
- **周度趋势报告**：7 天体重折线图、热量缺口柱状图、蛋白质达成率图、步数趋势图、每日明细表、AI 洞察

模板内含示例数据，AI 使用时只需替换 `<script type="application/json">` 标签内的 JSON 即可自动渲染。

## 数据保护 — 三道防线

- **单日异常拦截**：体重日波动 >1.5kg、步数异常、摄入 <800 或 >4000 kcal 时标注警告
- **多日趋势预警**：连续 3 天蛋白不足、连续 5 天大赤字、连续 3 天睡眠 <6h 时触发预警
- **跨指标交叉验证**：体重↓+腰围不变 → 水重变化；体重不变+腰围↓ → 身体重组信号
